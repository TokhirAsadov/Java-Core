# Java'da Operator Precedence

Java'da **Operator Precedence** (operatorlarning ustuvorligi) - bu ifodada bir nechta operatorlar mavjud bo'lganda, qaysi operator birinchi bajarilishini belgilaydigan qoidalar to'plamidir. Operatorlarning ustuvorligi va ularning bir-biriga nisbatan qanday tartibda ishlashi Java dasturlash tilida aniq belgilangan. Agar operatorlarning ustuvorligi bir xil bo'lsa, ularning **assotsiativligi** (chapdan o'ngga yoki o'ngdan chapga ishlash) ham hisobga olinadi.

## Operator Precedence Tartibi
Quyida Java'da operatorlarning ustuvorligi yuqoridan pastga (eng yuqori ustuvorlikdan eng pastgacha) keltirilgan:

| **Ustuvorlik darajasi** | **Operatorlar** | **Ta'rifi** | **Assotsiativlik** |
|-------------------------|----------------|-------------|-------------------|
| 1 | `()` `[]` `.` | Qavslar, massiv indekslari, nuqta operatori | Chapdan o'ngga |
| 2 | `++` `--` | Postfix increment/decrement | Chapdan o'ngga |
| 3 | `++` `--` `+` `-` `!` `~` | Prefix increment/decrement, unary plus/minus, mantiqiy INKOR, bitli INKOR | O'ngdan chapga |
| 4 | `(type)` | Cast (turi o'zgartirish) | O'ngdan chapga |
| 5 | `*` `/` `%` | Ko'paytirish, bo'lish, modul | Chapdan o'ngga |
| 6 | `+` `-` | Qo'shish, ayirish | Chapdan o'ngga |
| 7 | `<<` `>>` `>>>` | Chapga siljish, o'ngga siljish, imzosiz o'ngga siljish | Chapdan o'ngga |
| 8 | `<` `<=` `>` `>=` | Kichik, kichik yoki teng, katta, katta yoki teng | Chapdan o'ngga |
| 9 | `==` `!=` | Tenglik, teng emas | Chapdan o'ngga |
| 10 | `&` | Bitli VA | Chapdan o'ngga |
| 11 | `^` | Bitli XOR | Chapdan o'ngga |
| 12 | `|` | Bitli YOKI | Chapdan o'ngga |
| 13 | `&&` | Mantiqiy VA | Chapdan o'ngga |
| 14 | `||` | Mantiqiy YOKI | Chapdan o'ngga |
| 15 | `?:` | Shartli (ternary) operator | O'ngdan chapga |
| 16 | `=` `+=` `-=` `*=` `/=` `%=` `&=` `^=` `|=` `<<=` `>>=` `>>>=` | Tayinlash operatorlari | O'ngdan chapga |

## Asosiy Qoidalar
1. **Yuqori ustuvorlikdagi operatorlar birinchi bajariladi**: Masalan, `*` va `/` operatorlari `+` va `-` dan oldin bajariladi.
2. **Bir xil ustuvorlikdagi operatorlar**: Agar operatorlarning ustuvorligi teng bo'lsa, assotsiativlik qoidasi qo'llaniladi (ko'pincha chapdan o'ngga, lekin ba'zi hollarda o'ngdan chapga).
3. **Qavslar (`()`)**: Qavslar eng yuqori ustuvorlikka ega va operatorlar tartibini o'zgartirish uchun ishlatiladi. Qavslar ichidagi ifoda birinchi navbatda hisoblanadi.

## Misollar

### Misol 1: Arifmetik operatorlar
```java
int x = 2 + 3 * 4; // * birinchi bajariladi
System.out.println(x); // Natija: 14 (3 * 4 = 12, keyin 2 + 12 = 14)
```
- **Tushuntirish**: `*` operatori `+` dan yuqori ustuvorlikka ega, shuning uchun `3 * 4` avval hisoblanadi.

### Misol 2: Qavslar bilan
```java
int x = (2 + 3) * 4; // () birinchi bajariladi
System.out.println(x); // Natija: 20 (2 + 3 = 5, keyin 5 * 4 = 20)
```
- **Tushuntirish**: Qavslar ustuvorlikni o'zgartiradi, shuning uchun `2 + 3` avval hisoblanadi.

### Misol 3: Mantiqiy operatorlar
```java
boolean a = true, b = false, c = true;
boolean result = a || b && c; // && birinchi, keyin || bajariladi
System.out.println(result); // Natija: true (b && c = false, keyin a || false = true)
```
- **Tushuntirish**: `&&` operatori `||` dan yuqori ustuvorlikka ega.

### Misol 4: Bir xil ustuvorlik va assotsiativlik
```java
int x = 10;
x *= 2 + 3; // += va *= o'ngdan chapga
System.out.println(x); // Natija: 50 (2 + 3 = 5, keyin x * 5 = 50)
```
- **Tushuntirish**: `2 + 3` avval hisoblanadi, keyin `*=` operatori o'ngdan chapga ishlaydi.

### Misol 5: Murakkab ifoda
```java
int a = 5, b = 2, c = 3;
int result = a + b * c - a / b; // * va / birinchi, keyin + va -
System.out.println(result); // Natija: 9 (b * c = 6, a / b = 2, keyin 5 + 6 - 2 = 9)
```
- **Tushuntirish**: `*` va `/` bir xil ustuvorlikka ega, shuning uchun chapdan o'ngga bajariladi.

## E'tibor Beriladigan Jihatlar
- **Qavslarning ahamiyati**: Murakkab ifodalarda aniqlik uchun har doim qavslar ishlatish tavsiya etiladi, bu kodni o'qilishi oson va xatosiz qiladi.
- **Assotsiativlik**: Tayinlash (`=`) va shartli (`?:`) operatorlari o'ngdan chapga ishlaydi, aksariyat operatorlar esa chapdan o'ngga.
- **Xatolar oldini olish**: Noto'g'ri ustuvorlik tufayli kutilmagan natijalar chiqishi mumkin, shuning uchun operatorlar tartibini yaxshi tushunish muhim.

## Xulosa
Java'da operator precedence dastur ifodalarining to'g'ri va kutilgan tartibda bajarilishini ta'minlaydi. Eng yuqori ustuvorlik qavslardan boshlanadi va tayinlash operatorlarida tugaydi. Qavslar yordamida ustuvorlikni o'zgartirish mumkin, bu esa kodni o'qilishi oson va xatosiz qiladi.
