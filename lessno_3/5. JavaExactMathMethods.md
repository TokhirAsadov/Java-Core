# Java'da Xavfsiz Arifmetik Operatsiyalar (Exact Metodlari)

Java'da `Math` classi matematik operatsiyalarni xavfsizroq qilish uchun bir qator metodlarni taqdim etadi, ayniqsa butun sonlar (`int` va `long`) bilan arifmetik hisob-kitoblar paytida **to'lib ketish** (overflow) muammolarini aniqlash va oldini olish uchun. Oddiy matematik operatorlar (`+`, `-`, `*`, `/`) to'lib ketish sodir bo'lganda noto'g'ri natija qaytaradi va hech qanday xato xabarisiz ishlashni davom ettiradi. Masalan, `1000000000 * 3` hisoblanishi `-1294967296` kabi noto'g'ri natija beradi, chunki `int` turining chegarasi (2^31 - 1, ya'ni taxminan 2.147 milliard) oshib ketadi. `Math` classidagi **Exact** metodlar bunday hollarda xatolarni aniqlab, `ArithmeticException` xato xabarini chiqaradi.

## 1. To'lib Ketish (Overflow) Muammosi
Java'da butun son turlari cheklangan qiymat oralig'iga ega:
- **`int`**: -2^31 dan 2^31-1 gacha (-2,147,483,648 dan 2,147,483,647 gacha).
- **`long`**: -2^63 dan 2^63-1 gacha.

Agar hisob-kitob ushbu chegaralardan oshsa, natija noto'g'ri bo'ladi, chunki qiymat oralig'ida "aylana" hosil qiladi (wrap-around). 

**Misol:**
```java
int a = 1000000000; // 1 milliard
int result = a * 3; // To'lib ketish sodir bo'ladi
System.out.println(result); // Natija: -1294967296 (noto'g'ri)
```

## 2. Math classidagi Exact Metodlari
Java 8 va undan keyingi versiyalarda `Math` classi to'lib ketishni aniqlash uchun quyidagi **Exact** metodlarni taqdim etadi. Agar hisob-kitob natijasi ma'lumot turining chegarasidan oshsa, bu metodlar `ArithmeticException` chiqaradi.

### 2.1. Math.multiplyExact
- **Ta'rifi**: Ikki sonni ko'paytiradi va natija chegaradan oshsa, `ArithmeticException` chiqaradi.
- **Imzolar**:
  - `Math.multiplyExact(int a, int b)` → `int`
  - `Math.multiplyExact(long a, long b)` → `long`
  - `Math.multiplyExact(long a, int b)` → `long` (Java 9+)
- **Misol:**
```java
try {
    int result = Math.multiplyExact(1000000000, 3);
    System.out.println(result);
} catch (ArithmeticException e) {
    System.out.println("Xato: To'lib ketish sodir bo'ldi!");
}
// Natija: Xato: To'lib ketish sodir bo'ldi!
```
- **Tushuntirish**: `1000000000 * 3` `int` chegarasidan oshadi, shuning uchun `ArithmeticException` chiqadi.

### 2.2. Math.addExact
- **Ta'rifi**: Ikki sonni qo'shadi va natija chegaradan oshsa, `ArithmeticException` chiqaradi.
- **Imzolar**:
  - `Math.addExact(int a, int b)` → `int`
  - `Math.addExact(long a, long b)` → `long`
- **Misol:**
```java
try {
    int result = Math.addExact(Integer.MAX_VALUE, 1);
    System.out.println("Qo'shish: " + result);
} catch (ArithmeticException e) {
    System.out.println("Xato: To'lib ketish sodir bo'ldi!");
}
// Natija: Xato: To'lib ketish sodir bo'ldi!
```
- **Tushuntirish**: `Integer.MAX_VALUE` (2,147,483,647) ga 1 qo'shilsa, `int` chegarasi oshadi.

### 2.3. Math.subtractExact
- **Ta'rifi**: Bir sondan ikkinchisini ayiradi va natija chegaradan oshsa (masalan, manfiy chegaradan pastga tushsa), `ArithmeticException` chiqaradi.
- **Imzolar**:
  - `Math.subtractExact(int a, int b)` → `int`
  - `Math.subtractExact(long a, long b)` → `long`
- **Misol:**
```java
try {
    int result = Math.subtractExact(Integer.MIN_VALUE, 1);
    System.out.println("Ayirish: " + result);
} catch (ArithmeticException e) {
    System.out.println("Xato: To'lib ketish sodir bo'ldi!");
}
// Natija: Xato: To'lib ketish sodir bo'ldi!
```
- **Tushuntirish**: `Integer.MIN_VALUE` (-2,147,483,648) dan 1 ayirilsa, `int` chegarasi pastga tushadi.

### 2.4. Math.incrementExact
- **Ta'rifi**: Berilgan sonni 1 ga oshiradi va natija chegaradan oshsa, `ArithmeticException` chiqaradi.
- **Imzolar**:
  - `Math.incrementExact(int a)` → `int`
  - `Math.incrementExact(long a)` → `long`
- **Misol:**
```java
try {
    int result = Math.incrementExact(Integer.MAX_VALUE);
    System.out.println("Increment: " + result);
} catch (ArithmeticException e) {
    System.out.println("Xato: To'lib ketish sodir bo'ldi!");
}
// Natija: Xato: To'lib ketish sodir bo'ldi!
```

### 2.5. Math.decrementExact
- **Ta'rifi**: Berilgan sonni 1 ga kamaytiradi va natija chegaradan pastga tushsa, `ArithmeticException` chiqaradi.
- **Imzolar**:
  - `Math.decrementExact(int a)` → `int`
  - `Math.decrementExact(long a)` → `long`
- **Misol:**
```java
try {
    int result = Math.decrementExact(Integer.MIN_VALUE);
    System.out.println("Decrement: " + result);
} catch (ArithmeticException e) {
    System.out.println("Xato: To'lib ketish sodir bo'ldi!");
}
// Natija: Xato: To'lib ketish sodir bo'ldi!
```

### 2.6. Math.negateExact
- **Ta'rifi**: Berilgan sonning ishorasini o'zgartiradi (musbatdan manfiyga yoki aksincha) va natija chegaradan oshsa, `ArithmeticException` chiqaradi.
- **Imzolar**:
  - `Math.negateExact(int a)` → `int`
  - `Math.negateExact(long a)` → `long`
- **Misol:**
```java
try {
    int result = Math.negateExact(Integer.MIN_VALUE);
    System.out.println("Negate: " + result);
} catch (ArithmeticException e) {
    System.out.println("Xato: To'lib ketish sodir bo'ldi!");
}
// Natija: Xato: To'lib ketish sodir bo'ldi!
```
- **Tushuntirish**: `Integer.MIN_VALUE` (-2,147,483,648) ishorasi o'zgartirilsa, musbat qiymat `int` chegarasidan oshadi.

### 2.7. Math.toIntExact
- **Ta'rifi**: `long` turidagi qiymatni `int` turiga aylantiradi, agar qiymat `int` chegarasidan oshsa, `ArithmeticException` chiqaradi.
- **Imzo**: `Math.toIntExact(long value)` → `int`
- **Misol:**
```java
try {
    long bigValue = 2147483648L; // int chegarasidan katta
    int result = Math.toIntExact(bigValue);
    System.out.println("long'dan int'ga: " + result);
} catch (ArithmeticException e) {
    System.out.println("Xato: To'lib ketish sodir bo'ldi!");
}
// Natija: Xato: To'lib ketish sodir bo'ldi!
```

## 3. Foydalanish Holatlari
- **Xavfsiz arifmetika**: Katta hisob-kitoblar (masalan, moliyaviy operatsiyalar) uchun to'lib ketishni oldini olish.
- **Dastur ishonchliligi**: Noto'g'ri natijalar o'rniga xatolarni aniqlash va ularni qayta ishlash.
- **Ma'lumotlar konvertatsiyasi**: `long` dan `int` ga xavfsiz o'tish uchun `Math.toIntExact`.

## 4. Amaliy Misol (To'liq dastur)
```java
public class ExactMathExample {
    public static void main(String[] args) {
        // multiplyExact
        try {
            int result = Math.multiplyExact(1000000000, 3);
            System.out.println("Ko'paytirish: " + result);
        } catch (ArithmeticException e) {
            System.out.println("multiplyExact xatosi: To'lib ketish sodir bo'ldi!");
        }

        // addExact
        try {
            int result = Math.addExact(Integer.MAX_VALUE, 1);
            System.out.println("Qo'shish: " + result);
        } catch (ArithmeticException e) {
            System.out.println("addExact xatosi: To'lib ketish sodir bo'ldi!");
        }

        // subtractExact
        try {
            int result = Math.subtractExact(Integer.MIN_VALUE, 1);
            System.out.println("Ayirish: " + result);
        } catch (ArithmeticException e) {
            System.out.println("subtractExact xatosi: To'lib ketish sodir bo'ldi!");
        }

        // toIntExact
        try {
            long bigValue = 2147483648L;
            int result = Math.toIntExact(bigValue);
            System.out.println("long'dan int'ga: " + result);
        } catch (ArithmeticException e) {
            System.out.println("toIntExact xatosi: To'lib ketish sodir bo'ldi!");
        }
    }
}
```
**Natija:**
```
multiplyExact xatosi: To'lib ketish sodir bo'ldi!
addExact xatosi: To'lib ketish sodir bo'ldi!
subtractExact xatosi: To'lib ketish sodir bo'ldi!
toIntExact xatosi: To'lib ketish sodir bo'ldi!
```

## 5. E'tibor Beriladigan Jihatlar
- **Exception ishlash**: Exact metodlarni ishlatganda har doim `try-catch` bloki bilan ishlatish tavsiya etiladi, chunki `ArithmeticException` chiqishi mumkin.
- **Faqat butun sonlar uchun**: Bu metodlar faqat `int` va `long` turlari bilan ishlaydi. `float` yoki `double` uchun to'lib ketishni aniqlash imkoni yo'q.
- **Samaradorlik**: Exact metodlar oddiy operatorlarga nisbatan biroz sekinroq bo'lishi mumkin, chunki ular chegaralarni tekshiradi.
- **Alternativa**: Agar to'lib ketish muammo bo'lmasa, oddiy operatorlar (`+`, `-`, `*`) ishlatilishi mumkin, lekin xavfsizlik uchun Exact metodlar afzalroq.

## Xulosa
`Math` classidagi Exact metodlari (`multiplyExact`, `addExact`, `subtractExact`, `incrementExact`, `decrementExact`, `negateExact`, `toIntExact`) butun sonlar bilan ishlashda to'lib ketish muammolarini aniqlash va oldini olish uchun mo'ljallangan. Ular oddiy operatorlardan farqli o'laroq, noto'g'ri natija qaytarish o'rniga `ArithmeticException` chiqaradi, bu esa dastur ishonchliligini oshiradi. Ushbu metodlar moliyaviy hisob-kitoblar, katta ma'lumotlar bilan ishlash yoki xavfsiz konvertatsiya talab qilinadigan holatlarda juda foydali.
