# Java'da StrictMath classi

Java'da `java.lang.StrictMath` classi `java.lang.Math` classiga o'xshash matematik funksiyalarni taqdim etadi, lekin u eng yuqori darajada **natijalarning aniqligi va platformadan mustaqilligi**ni ta'minlashga mo'ljallangan. `Math` classi kompyuterning floating-point birligi (FPU) dan foydalanib, eng yuqori ishlash tezligini ta'minlasa, `StrictMath` classi barcha platformalarda bir xil natijalarni kafolatlash uchun **Freely Distributable Math Library (fdlibm)** algoritmlariga asoslanadi.

## Math va StrictMath O'rtasidagi Farqlar
1. **Ishlash Tezligi**:
   - `Math` classi: Kompyuterning FPU dan foydalanadi, bu esa yuqori ishlash tezligini ta'minlaydi, lekin natijalar platformaga qarab ozgina farq qilishi mumkin (masalan, turli protsessorlar yoki operatsion tizimlarda).
   - `StrictMath` classi: `fdlibm` algoritmlariga qat'iy rioya qiladi, bu esa ishlash tezligini biroz pasaytiradi, lekin natijalar barcha platformalarda bir xil bo'ladi.

2. **Natijalarning Aniqligi**:
   - `Math` classi: Ba'zi hollarda protsessorning FPU xususiyatlariga qarab natijalar minimal farqlar bilan chiqishi mumkin (masalan, `double` qiymatlardagi aniqlik farqlari).
   - `StrictMath` classi: Har bir platformada bir xil natijani kafolatlaydi, chunki u qat'iy va standartlashtirilgan algoritmlardan foydalanadi.

3. **Foydalanish Holatlari**:
   - `Math`: Tezlik muhim bo'lgan va kichik natija farqlari muammo bo'lmagan ilovalar uchun (masalan, grafikalar, o'yinlar).
   - `StrictMath`: Natijalarning aniqligi va platformadan mustaqilligi muhim bo'lgan ilovalar uchun (masalan, ilmiy hisob-kitoblar, moliyaviy tizimlar yoki standartlashtirilgan testlar).

## StrictMath classining Xususiyatlari
- **Statik Metodlar**: `StrictMath` classidagi barcha metodlar statik bo'lib, `Math` classiga o'xshab ob'ekt yaratmasdan ishlatiladi (masalan, `StrictMath.sin(x)`).
- **Metodlar**: `StrictMath` classi `Math` classidagi deyarli barcha metodlarni takrorlaydi:
  - **Arifmetik**: `abs`, `max`, `min`
  - **Yaxlitlash**: `round`, `floor`, `ceil`, `rint`
  - **Eksponensial va logarifmik**: `exp`, `log`, `log10`, `pow`, `sqrt`, `cbrt`
  - **Trigonometrik**: `sin`, `cos`, `tan`, `asin`, `acos`, `atan`, `atan2`
  - **Boshqalar**: `toRadians`, `toDegrees`, `hypot`
- **Konstantalar**: `StrictMath` ham `Math.PI` va `Math.E` konstantalarini taqdim etadi, ular `Math` classidagi bilan bir xil.
- **Exact Metodlar**: `StrictMath` classi `Math` classidagi kabi `multiplyExact`, `addExact`, `subtractExact`, `incrementExact`, `decrementExact`, `negateExact`, va `toIntExact` metodlarini qo'llab-quvvatlaydi.

## Math.random() Farqi
- `Math.random()` tasodifiy sonlar generatori sifatida ishlaydi va platformaga qarab farqli ketma-ketliklar chiqarishi mumkin.
- `StrictMath` classida `random()` metodi yo'q, chunki tasodifiy sonlar generatsiyasi platformadan mustaqil bo'lishini kafolatlash qiyin. Agar platformadan mustaqil tasodifiy sonlar kerak bo'lsa, `java.util.Random` yoki `java.security.SecureRandom` classidan foydalanish tavsiya etiladi.

## Misol: Math va StrictMath Taqqoslash
```java
public class MathVsStrictMath {
    public static void main(String[] args) {
        double angle = Math.toRadians(90);

        // Math classi
        System.out.println("Math.sin(90°): " + Math.sin(angle)); // Natija: 1.0
        System.out.println("Math.sqrt(16): " + Math.sqrt(16));   // Natija: 4.0

        // StrictMath classi
        System.out.println("StrictMath.sin(90°): " + StrictMath.sin(angle)); // Natija: 1.0
        System.out.println("StrictMath.sqrt(16): " + StrictMath.sqrt(16));   // Natija: 4.0

        // Exact metodlar
        try {
            int result = StrictMath.multiplyExact(1000000000, 3);
            System.out.println("StrictMath.multiplyExact: " + result);
        } catch (ArithmeticException e) {
            System.out.println("StrictMath.multiplyExact xatosi: To'lib ketish sodir bo'ldi!");
        }
    }
}
```
**Natija:**
```
Math.sin(90°): 1.0
Math.sqrt(16): 4.0
StrictMath.sin(90°): 1.0
StrictMath.sqrt(16): 4.0
StrictMath.multiplyExact xatosi: To'lib ketish sodir bo'ldi!
```

## E'tibor Beriladigan Jihatlar
- **Tezlik va Aniqlik**: `Math` classi tezlikni birinchi o'ringa qo'ysa, `StrictMath` aniqlik va platformadan mustaqillikni birinchi o'ringa qo'yadi.
- **Foydalanish**: Agar dasturingiz turli platformalarda bir xil natija chiqarishi muhim bo'lsa (masalan, ilmiy dasturlar yoki standartlashtirilgan testlar), `StrictMath` ishlatish afzal. Aks holda, `Math` classi ko'proq ishlatiladi, chunki u tezroq.
- **Radianlar**: `StrictMath` ham trigonometrik funksiyalar uchun radianlardan foydalanadi, shuning uchun `toRadians` yoki `toDegrees` metodlari kerak bo'ladi.
- **Exact Metodlar**: `StrictMath` classidagi Exact metodlar `Math` classidagi bilan bir xil xatti-harakatlarga ega, ya'ni to'lib ketishni aniqlashda bir xil `ArithmeticException` chiqaradi.

## Foydalanish Holatlari
- **Ilmiy Hisob-kitoblar**: Turli platformalarda bir xil natija talab qilinganda (masalan, fizika yoki muhandislik simulyatsiyalari).
- **Standartlashtirilgan Testlar**: Test natijalarining barcha muhitlarda bir xil bo'lishi zarur bo'lgan hollarda.
- **Moliyaviy Tizimlar**: Aniqlik va izchillik muhim bo'lgan joylarda.

## Xulosa
`StrictMath` classi `Math` classiga muqobil sifatida platformadan mustaqil va aniq natijalarni ta'minlash uchun ishlatiladi. U `fdlibm` algoritmlariga asoslanib, barcha platformalarda bir xil natijalarni kafolatlaydi, lekin ishlash tezligi jihatidan `Math` classiga nisbatan biroz sekinroq bo'lishi mumkin. Agar dasturda tezlikdan ko'ra natijalarning izchilligi muhim bo'lsa, `StrictMath` afzalroqdir.
