# Java naming conventions

Java’da **naming conventions** dasturchilar tomonidan o‘zgaruvchilar, metodlar, class'lar va boshqa elementlarga nom berishda qo‘llaniladigan standart usullardir. Bu uslublar kodning o‘qilishi, tushunilishi va jamoaviy ishlashni osonlashtiradi. Quyida camelCase’dan boshlab nomlash uslublari va ulardagi xatolar haqida ma'lumot beriladi.

## 1. camelCase
- **Ta’rif**: So‘zlar bir-biriga ulanadi, birinchi so‘zning barcha harflari kichik, keyingi so‘zlarning birinchi harfi katta, qolgan harflari kichik yoziladi. So‘zlar orasida bo‘shliq yoki ajratuvchi belgi ishlatilmaydi.
- **Qo‘llanilishi**: Java’da o‘zgaruvchilar va metod nomlari uchun eng ko‘p ishlatiladi. Boshqa tillarda (JavaScript, Python) ham keng tarqalgan.
- **Misol**:
  ```java
  int userAge = 25; // o‘zgaruvchi
  String firstName = "Ali"; // o‘zgaruvchi
  void calculateTotalPrice() { ... } // metod
  ```
- **Yana misollar**:
  - `totalAmount`
  - `isActive`
  - `getUserDetails`

## 2. PascalCase
- **Ta’rif**: Har bir so‘zning birinchi harfi katta, qolgan harflari kichik yoziladi. So‘zlar orasida bo‘shliq yoki ajratuvchi belgi ishlatilmaydi. camelCase’dan farqi – birinchi so‘zning ham birinchi harfi katta.
- **Qo‘llanilishi**: Java’da class' (class) va interfeys (interface) nomlari uchun ishlatiladi. Boshqa tillarda (C#, .NET) ham keng qo‘llaniladi.
- **Misol**:
  ```java
  public class UserProfile { ... } // class'
  public interface Printable { ... } // interfeys
  public enum ErrorCode { ... } // enum
  ```
- **Yana misollar**:
  - `BankAccount`
  - `CustomerOrder`
  - `DataProcessor`

## 3. snake_case
- **Ta’rif**: So‘zlar kichik harflar bilan yoziladi va ular orasida pastki chiziq (`_`) bilan ajratiladi.
- **Qo‘llanilishi**: Python va Ruby kabi tillarda o‘zgaruvchi va metod nomlari uchun keng tarqalgan. Java’da kamroq ishlatiladi, lekin ba’zi hollarda (masalan, test nomlari yoki fayl nomlari) qo‘llanilishi mumkin.
- **Misol**:
  ```java
  int total_amount = 100; // Java’da kam ishlatiladi
  String user_name = "Vali";
  void calculate_total_price() { ... }
  ```
- **Yana misollar**:
  - `max_retries`
  - `file_path`
  - `connection_timeout`

## 4. kebab-case
- **Ta’rif**: So‘zlar kichik harflar bilan yoziladi va ular orasida defis (`-`) bilan ajratiladi.
- **Qo‘llanilishi**: Java’da deyarli ishlatilmaydi, chunki defis (`-`) identifikator nomlarida ruxsat etilmagan belgi. Ammo CSS, HTML yoki fayl nomlarida keng tarqalgan. Java’da kebab-case fayl nomlari yoki resurs nomlari uchun ishlatilishi mumkin.
- **Misol** (Java’da to‘g‘ridan-to‘g‘ri ishlatilmaydi, lekin fayl nomlari uchun):
  ```
  config-file.properties
  user-profile-data.json
  ```
- **Yana misollar**:
  - `home-page`
  - `data-source`
  - `error-log`

## 5. MACRO_CASE (SCREAMING_SNAKE_CASE)
- **Ta’rif**: Barcha harflar katta yoziladi va so‘zlar orasida pastki chiziq (`_`) bilan ajratiladi. “Screaming” deb ataladi, chunki barcha harflar katta.
- **Qo‘llanilishi**: Java’da doimiy o‘zgaruvchilar (constants, `final` bilan e’lon qilingan) va enum qiymatlari uchun ishlatiladi. C va C++ kabi tillarda ham keng tarqalgan.
- **Misol**:
  ```java
  public static final int MAX_USERS = 100; // doimiy
  public static final double PI = 3.14159; // doimiy
  public enum Status { ACTIVE, INACTIVE } // enum qiymatlari
  ```
- **Yana misollar**:
  - `DEFAULT_TIMEOUT`
  - `TAX_RATE`
  - `ERROR_CODE`

## 6. Train-Case
- **Ta’rif**: Har bir so‘zning birinchi harfi katta, qolgan harflari kichik yoziladi va so‘zlar defis (`-`) bilan ajratiladi. PascalCase’ga o‘xshaydi, lekin defis bilan ajratiladi.
- **Qo‘llanilishi**: Java’da to‘g‘ridan-to‘g‘ri ishlatilmaydi, chunki defis identifikator nomlarida ruxsat etilmagan. Ammo fayl nomlari, API yo‘llari yoki boshqa tashqi resurslarda ishlatilishi mumkin (masalan, REST API URL’larida).
- **Misol** (Java’da to‘g‘ridan-to‘g‘ri emas, lekin masalan, fayl nomlari):
  ```
  User-Profile-Manager.java
  Data-Processing-Unit.config
  ```
- **Yana misollar**:
  - `Order-Details`
  - `User-Authentication`
  - `Payment-Processor`

## Java’dagi asosiy qo‘llanilishi
Java’da nomlash uslublari quyidagicha taqsimlanadi:
- **camelCase**: O‘zgaruvchilar va metodlar uchun (`userName`, `calculateTotal`).
- **PascalCase**: class'lar, interfeyslar va enumlar uchun (`UserProfile`, `Printable`).
- **MACRO_CASE**: Doimiy o‘zgaruvchilar va enum qiymatlari uchun (`MAX_USERS`, `ACTIVE`).
- **snake_case**: Java’da kam ishlatiladi, lekin test nomlari yoki fayllarda ko‘rinishi mumkin (`test_user_login`).
- **kebab-case** va **Train-Case**: Java kodida ishlatilmaydi, lekin fayl nomlari yoki tashqi resurslarda qo‘llaniladi.

## Misol: Barcha uslublarni bir dasturda ko‘rsatish
Java’da faqat ruxsat etilgan uslublarni (camelCase, PascalCase, MACRO_CASE) ishlatish mumkin, shuning uchun quyidagi misol shu uslublarni ko‘rsatadi:

```java
package com.example.app; // snake_case (paket nomlari uchun kichik harflar)

public class UserProfile { // PascalCase (class')
    public static final int MAX_USERS = 100; // MACRO_CASE (doimiy)
    private int userAge; // camelCase (o‘zgaruvchi)
    private String firstName; // camelCase (o‘zgaruvchi)

    public void setUserAge(int age) { // camelCase (metod)
        this.userAge = age;
    }

    public String getFirstName() { // camelCase (metod)
        return firstName;
    }

    public enum UserStatus { // PascalCase (enum)
        ACTIVE, INACTIVE // MACRO_CASE (enum qiymatlari)
    }
}
```

Fayl nomi misoli (kebab-case yoki Train-Case):
```
user-profile.properties // kebab-case
User-Profile-Config.java // Train-Case
```

**Tushuntirish**:
- `com.example.app`: Paket nomi snake_case usulida.
- `UserProfile`: class' nomi PascalCase usulida.
- `MAX_USERS`: Doimiy o‘zgaruvchi MACRO_CASE usulida.
- `userAge`, `firstName`, `setUserAge`, `getFirstName`: O‘zgaruvchi va metodlar camelCase usulida.
- `UserStatus`, `ACTIVE`, `INACTIVE`: Enum PascalCase va enum qiymatlari MACRO_CASE usulida.
- Fayl nomlari: `user-profile.properties` (kebab-case) va `User-Profile-Config.java` (Train-Case).

## Qo‘shimcha Maslahatlar
- **Java uchun moslik**: Java’da faqat camelCase, PascalCase va MACRO_CASE identifikatorlar uchun ishlatiladi, chunki defis (`-`) ruxsat etilmagan.
- **Aniqlik**: Nomlar elementning maqsadini aniq ifodalashi kerak (masalan, `userAge` yoki `calculateTotal`).
- **Bir xillik**: Loyiha bo‘yicha bir xil uslubdan foydalaning, bu jamoaviy ishlashda chalkashlikni kamaytiradi.
- **Fayl nomlari**: Java class' fayllari odatda PascalCase’da bo‘ladi (masalan, `UserProfile.java`), lekin tashqi resurslar uchun kebab-case yoki Train-Case ishlatilishi mumkin.

## Xatolar va Ulardan Saqlanish
- **Noto‘g‘ri uslub**: `public class userprofile` (PascalCase o‘rniga kichik harflar).
- **Noaniq nomlar**: `int a` o‘rniga `int userAge`.
- **Ruxsat etilmagan belgilar**: Java’da `user-name` (kebab-case) ishlatib bo‘lmaydi.
- **Aralash uslublar**: Bir loyihada camelCase va snake_case aralash ishlatishdan saqlaning.
